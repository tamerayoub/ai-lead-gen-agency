1) Create & configure a Facebook App

Go to developers.facebook.com → My Apps → Create App (type: Business).

Add the Messenger product to the app.

Put your Privacy Policy URL and Data Deletion Callback URL on the app (required for review/production).

Switch app to Live only after passing review (you can test in Development first).

2) Permissions you’ll need (Delegated via Facebook Login)

When a customer connects their Page inside your CRM, you’ll OAuth them and ask for:

pages_show_list – list pages the user manages

pages_manage_metadata – subscribe to webhooks

pages_messaging – send/receive messages

pages_read_engagement – read messages/comments/engagement

pages_manage_engagement – manage and reply

(Optional) pages_read_user_content if you’ll read more page content

App Review: pages_messaging and some others require App Review + sometimes Business Verification. You’ll submit a screencast showing how your CRM uses each permission.

3) Tokens (how you actually send messages)

User logs in → you get a short-lived user access token.

Exchange for a long-lived user token.

Use that to fetch the user’s Pages → select a Page → get a Page Access Token.

Store the Page Access Token per tenant (encrypted). That’s what you’ll use for Send API calls.

4) Webhooks (to receive messages)

Expose an HTTPS endpoint in your CRM backend, e.g.:

POST /webhooks/facebook/messenger

In the app’s Messenger → Settings, set Callback URL + Verify Token.

Facebook will send a GET verify request (hub.mode, hub.verify_token, hub.challenge).

Reply with the hub.challenge if the token matches.

Subscribe the Page to your app:

POST /{page-id}/subscribed_apps with the page token.

Subscribe to events: messages, messaging_postbacks, message_deliveries, message_reads, etc.

Verify requests via X-Hub-Signature-256 HMAC header (security).

5) Send/receive APIs you’ll actually call

Send API: POST https://graph.facebook.com/v{VERSION}/me/messages

Body: { recipient: { id: <PSID> }, message: { text: "Hello!" } }

Auth: Page Access Token (querystring access_token or header)

List Pages: GET /me/accounts (with user token)

Subscribe Page: POST /{page-id}/subscribed_apps

You’ll map the PSID (Page-scoped ID) to your CRM Contact/Lead record.

6) Product rules you must respect

24-hour messaging window: after a user messages your Page, you have 24h to send any content.

Outside 24h, you must use message tags (e.g., CONFIRMED_EVENT_UPDATE, ACCOUNT_UPDATE, POST_PURCHASE_UPDATE) or Sponsored Messages (paid). Pure promotions outside the 24h window are not allowed.

(Optional) Handover Protocol if you juggle multiple inboxes/agents: pick a Primary Receiver and Secondary Receiver app.

7) Multi-tenant design (recommended)

OAuth connect flow per tenant:
UI: “Connect Facebook Page” → Facebook Login → choose Page → we store Page token & Page ID.

Token store: Encrypt Page tokens, scoped by tenant_id.

Webhook router: One global webhook endpoint → look up page_id in payload → resolve tenant_id → enqueue job.

Queue + worker: Process inbound messages → attach to the right Lead/Conversation.
Outbound messages go from worker → Send API (respect rate limits).

CRM mapping: Store PSID, page_id, thread timestamps, message direction, attachments.

8) Messenger Profile setup (nice to have)

Get Started payload, greeting text, Persistent Menu, Icebreakers.

Quick Replies, Buttons, Generic Templates (carousels), media attachments (upload first, then send).

9) Minimal Node/Express webhook skeleton
import express from "express";
import crypto from "crypto";
import fetch from "node-fetch";

const app = express();
app.use(express.json({ type: "*/*" })); // ensure we get raw body for signature

const FB_APP_SECRET = process.env.FB_APP_SECRET!;
const VERIFY_TOKEN = process.env.FB_VERIFY_TOKEN!;

// Verify webhook (GET)
app.get("/webhooks/facebook/messenger", (req, res) => {
  const mode = req.query["hub.mode"];
  const token = req.query["hub.verify_token"];
  const challenge = req.query["hub.challenge"];

  if (mode === "subscribe" && token === VERIFY_TOKEN) {
    return res.status(200).send(challenge);
  }
  return res.sendStatus(403);
});

// Optional: verify HMAC signature
function verifySignature(req: any) {
  const signature = req.headers["x-hub-signature-256"];
  if (!signature) return false;
  const payload = JSON.stringify(req.body);
  const hmac = crypto
    .createHmac("sha256", FB_APP_SECRET)
    .update(payload, "utf8")
    .digest("hex");
  return signature === `sha256=${hmac}`;
}

// Receive messages (POST)
app.post("/webhooks/facebook/messenger", async (req, res) => {
  // if (!verifySignature(req)) return res.sendStatus(403);

  const body = req.body;
  if (body.object !== "page") return res.sendStatus(404);

  for (const entry of body.entry || []) {
    const pageId = entry.id;
    for (const evt of entry.messaging || []) {
      const psid = evt.sender?.id;

// Send API helper (Page token must be fetched by tenant/page_id)
async function sendTextMessage(pageId: string, psid: string, text: string) {
  const pageAccessToken = await getPageTokenFor(pageId); // your token store
  const url = `https://graph.facebook.com/v19.0/me/messages?access_token=${pageAccessToken}`;
  const payload = { recipient: { id: psid }, message: { text } };
  const r = await fetch(url, { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify(payload) });
  if (!r.ok) console.error("Send API error", await r.text());
}
