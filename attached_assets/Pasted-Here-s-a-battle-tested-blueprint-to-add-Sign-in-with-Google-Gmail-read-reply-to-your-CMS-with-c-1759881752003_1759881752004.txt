Here’s a battle-tested blueprint to add “Sign in with Google + Gmail read/reply” to your CMS, with code patterns you can drop into Node/Express or ASP.NET Core.

1) Create your Google app & scopes

In Google Cloud Console, create a project → Enable the Gmail API.

Configure OAuth consent screen (app name, logo, domain, privacy policy).

Create OAuth 2.0 Client IDs (Web application). Add your CMS redirect URIs.

Scopes you’ll request

Use least privilege and add more via incremental auth only when needed:

Read mail: https://www.googleapis.com/auth/gmail.readonly

Send/reply: https://www.googleapis.com/auth/gmail.send

If you need labels/archiving: https://www.googleapis.com/auth/gmail.modify
These are restricted scopes and may trigger Google app verification (and sometimes a security assessment) before you can go to production for consumer accounts. Plan for this early.

2) Auth flow you’ll implement

Use the web-server OAuth 2.0 code flow (back-end exchanges the code for tokens). Ask for offline access to get a refresh token so your app can fetch mail and send replies when the user isn’t online. 
Google for Developers

High-level:

Frontend: “Sign in with Google” → send user to Google’s auth URL with your scopes.

Backend callback: exchange code → get {access_token, refresh_token, expires_in} and persist securely (encrypted, per user).

Use the Gmail API with the access token; refresh when expired.

3) Reading mail & replying (Gmail API)

List messages: users.messages.list (optionally with q search and labelIds), then users.messages.get to fetch headers/body.

Send a reply: build a full RFC 2822 message with headers (To, Subject, In-Reply-To, References) and set threadId so it stays in the same thread; base64url-encode it and call users.messages.send (or create a draft then drafts.send).

4) Keep the mailbox in sync (optional but recommended)

Use push notifications (Pub/Sub) + users.watch to be notified of changes. On notification, call users.history.list from the last stored historyId to figure out what changed, then fetch only those messages. (Expect some history IDs to yield no changes; just store the new historyId.)

Quick start: Node/Express (googleapis)
// npm i googleapis express cookie-session
import express from "express";
import { google } from "googleapis";

const app = express();
const oauth2 = new google.auth.OAuth2(
  process.env.GOOGLE_CLIENT_ID,
  process.env.GOOGLE_CLIENT_SECRET,
  "https://yourcms.com/oauth2/callback"
);

const SCOPES = [
  "https://www.googleapis.com/auth/gmail.readonly",
  "https://www.googleapis.com/auth/gmail.send"
];

app.get("/auth/google", (req, res) => {
  const url = oauth2.generateAuthUrl({
    access_type: "offline",
    prompt: "consent",         // ensures refresh_token on first consent
    scope: SCOPES
  });
  res.redirect(url);
});

app.get("/oauth2/callback", async (req, res) => {
  const { tokens } = await oauth2.getToken(req.query.code);
  // store tokens.refresh_token & tokens.access_token encrypted per user
  // e.g., in your Users table
  res.redirect("/inbox");
});

async function gmailClient(tokens) {
  const auth = new google.auth.OAuth2();
  auth.setCredentials(tokens); // include refresh_token for auto refresh
  return google.gmail({ version: "v1", auth });
}

// List messages
app.get("/api/messages", async (req, res) => {
  const gmail = await gmailClient(req.user.tokens);
  const list = await gmail.users.messages.list({ userId: "me", maxResults: 10 });
  res.json(list.data);
});

// Reply to a thread
app.post("/api/reply", express.json(), async (req, res) => {
  const { threadId, to, subject, body, inReplyTo, references } = req.body;
  const gmail = await gmailClient(req.user.tokens);

  const raw =
`To: ${to}
Subject: Re: ${subject}
In-Reply-To: ${inReplyTo}
References: ${references}
Content-Type: text/plain; charset="UTF-8"
MIME-Version: 1.0

${body}`;

  const encodedMessage = Buffer.from(raw).toString("base64").replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');
  const send = await gmail.users.messages.send({
    userId: "me",
    requestBody: { raw: encodedMessage, threadId }
  });

  res.json(send.data);
});


This follows Google’s guidance for sending (base64url in raw) and threading via threadId.

Quick start: ASP.NET Core (Gmail API .NET client)
// <PackageReference Include="Google.Apis.Gmail.v1" Version="*" />
// <PackageReference Include="Google.Apis.Auth" Version="*" />

// Startup: add Google auth middleware for the sign-in button,
// but use Google.Apis for Gmail API calls after you store tokens.

var scopes = new[]
{
    "https://www.googleapis.com/auth/gmail.readonly",
    "https://www.googleapis.com/auth/gmail.send"
};

// Exchange code for tokens (controller)
[HttpGet("/oauth2/callback")]
public async Task<IActionResult> GoogleCallback([FromQuery] string code)
{
    var flow = new GoogleAuthorizationCodeFlow(new GoogleAuthorizationCodeFlow.Initializer
    {
        ClientSecrets = new ClientSecrets
        {
            ClientId = Environment.GetEnvironmentVariable("GOOGLE_CLIENT_ID"),
            ClientSecret = Environment.GetEnvironmentVariable("GOOGLE_CLIENT_SECRET")
        },
        Scopes = scopes,
        AccessType = "offline",
        Prompt = "consent"
    });

    var token = await flow.ExchangeCodeForTokenAsync("me", code, "https://yourcms.com/oauth2/callback", CancellationToken.None);
    // Persist token.RefreshToken & token.AccessToken for this user (encrypted)
    return Redirect("/inbox");
}

// Use Gmail API
public async Task<IList<Message>> ListMessages(UserTokens tokens)
{
    var credential = GoogleCredential.FromAccessToken(tokens.AccessToken)
        .CreateScoped(scopes)
        .CreateWithUser("me");

    // If you’re storing refresh_token, use UserCredential for auto refresh.
    var service = new GmailService(new BaseClientService.Initializer
    {
        HttpClientInitializer = credential,
        ApplicationName = "Your CMS"
    });

    var req = service.Users.Messages.List("me");
    req.MaxResults = 10;
    var resp = await req.ExecuteAsync();
    return resp.Messages;
}

Data model & security checklist

Users table: google_user_id, email, access_token, refresh_token, expiry, scopes_granted, history_id (for sync), token_encrypted=true.

Encrypt tokens at rest (KMS/KeyVault) and rotate keys.

Least privilege scopes; use incremental auth (ask for gmail.send only when the user first tries to reply).

Common pitfalls (and fixes)

“raw” vs MIME parts confusion when sending: you must supply a single base64url-encoded RFC 2822 message in raw (don’t mix with JSON payload).
